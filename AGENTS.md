# pago-js — Agent Guide

`@pago-sh/sdk` — the official pago.sh SDK for TypeScript/JavaScript. **Generated by Speakeasy** from the pago.sh OpenAPI spec.

## ⚠️ Generated code
Most files are generated and **overwritten on regeneration** — do not hand-edit them. Source of truth: the OpenAPI spec + `.speakeasy/gen.yaml` (holds the SDK `version`). Durable customizations go in `.speakeasy/overlays`, custom hooks under `src/hooks/`, and `README.md`. Regeneration and publishing are **manual** (CI workflows were removed).

## Quick commands
```bash
pnpm install
pnpm build                     # tsc → dist/
pnpm lint
npm publish --access public    # publish @pago-sh/sdk to npm
```

## Structure
- `src/` — generated SDK (models, operations, hooks) · `.speakeasy/` — gen.yaml + overlays · `docs/` — usage snippets

## Git Workflow

Branch model — the same across every repo in the pago.sh ecosystem:

- **`main`** — production. Releases run from `main`. Never commit or push directly to `main`.
- **`develop`** — integration & testing. Validated here before reaching `main`.
- **`feat/<name>`** / **`fix/<name>`** — branch off `develop`, merge back into `develop`.

**Flow:** `feat/*` or `fix/*` → **`develop`** (test) → merge into **`main`** → production.
